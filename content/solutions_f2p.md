### Solutions to Families to Persons (revisited 2026 edition)

The Families to Persons case is about concurrent synchronization of heterogeneous models, it is an extension of the [2017 Families to persons case](2017/solutions_familiesToPersons.html).

- Paper: [The Families to Persons Case — Revisited](STAF_2026_paper_35.pdf)
- GitHub repository: [https://github.com/eMoflon/benchmarx](https://github.com/eMoflon/benchmarx)
- Issue Tracker: [https://github.com/eMoflon/benchmarx/issues](https://github.com/eMoflon/benchmarx/issues)

The following solutions have been submitted:

- [The Families to Persons Case Revisited: A BXtendDSL Solution with Concurrent Model Synchronization](STAF_2026_paper_110.pdf) by Thomas Buchmann
- [Handling Conflicting Changes in the Families to Persons Case with Synchronization Blocks](STAF_2026_paper_118.pdf) by Georg Hinkel
- [Let the Agent Sync: LLM-Driven Code Generation for the Families to Persons Case](STAF_2026_paper_119.pdf) by Thomas Buchmann
- [The BCerT Solution for Bidirectional Model-Driven Transformation](STAF_2026_paper_139.pdf) by Akram Idani
